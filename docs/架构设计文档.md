## 2. 核心接口定义代码

### 2.1 完整接口定义

参见 [`src/core/interfaces.ts`](../src/core/interfaces.ts)

```typescript
/**
 * 核心抽象接口层
 * 所有业务代码必须依赖这些接口，而不是直接依赖 Node.js 或 Skynet API
 */

/**
 * 日志接口 - 统一日志输出
 */
export interface ILogger {
  debug(message: string, ...args: any[]): void;
  info(message: string, ...args: any[]): void;
  warn(message: string, ...args: any[]): void;
  error(message: string, ...args: any[]): void;
}

/**
 * 定时器接口 - 统一时间操作
 */
export interface ITimer {
  setTimeout(ms: number, callback: () => void): any;
  clearTimeout(handle: any): void;
  sleep(ms: number): Promise<void>;      // 异步睡眠 ⭐
  now(): number;                          // 当前时间戳（秒）
}

/**
 * 网络接口 - 统一消息通信
 */
export interface INetwork {
  send(address: string, messageType: string, ...args: any[]): void;
  call(address: string, messageType: string, ...args: any[]): Promise<any>;  // ⭐
  dispatch(messageType: string, handler: (...args: any[]) => void | Promise<void>): void;
  ret(...args: any[]): void;
}

/**
 * 服务接口 - 统一服务管理
 */
export interface IService {
  start(callback: () => void | Promise<void>): void;  // ⭐
  exit(): void;
  newService(name: string, ...args: any[]): Promise<string>;
  self(): string;
  getenv(key: string): string | undefined;
  setenv(key: string, value: string): void;
}

/**
 * 运行时上下文 - 聚合所有接口
 */
export interface IRuntime {
  logger: ILogger;
  timer: ITimer;
  network: INetwork;
  service: IService;
  database?: IDatabase;  // 可选的数据库接口
}

/**
 * 全局运行时实例
 */
export let runtime: IRuntime;

export function setRuntime(rt: IRuntime): void {
  runtime = rt;
}
```

### 2.2 设计原则

#### ✅ 抽象原则

1. **依赖倒置**: 业务代码依赖接口，不依赖具体实现
2. **接口隔离**: 每个接口职责单一
3. **开闭原则**: 易于扩展新的适配器

#### 🔑 关键设计点

**异步方法返回 Promise:**
```typescript
// ✅ 正确：返回 Promise
sleep(ms: number): Promise<void>;
call(address: string, ...): Promise<any>;

// ❌ 错误：同步返回
sleep(ms: number): void;  // 无法在两个环境统一
```

这样设计的原因:
- **Node.js**: 原生支持 Promise
- **Skynet**: TSTL 会将 Promise 转换为协程 yield/resume

## 3. 异步处理实现方案

这是本框架最核心的技术难点！

### 3.1 问题分析

| 环境 | 异步模型 | API 示例 |
|-----|---------|----------|
| Node.js | Promise/async-await | `await fetch()` |
| Skynet | Lua Coroutine | `skynet.call()` 内部 yield |

**挑战**: 如何让开发者统一使用 `async/await`，同时:
- Node.js 侧映射为 Promise
- Skynet 侧转换为 Coroutine

### 3.2 解决方案

#### 方案架构

```
┌──────────────────────────────────────┐
│   TypeScript 业务代码                 │
│   async function login() {           │
│     await network.call(...)          │
│   }                                  │
└──────────┬───────────────────────────┘
           │
    ┌──────┴───────┐
    │              │
    ▼              ▼
┌─────────┐  ┌──────────┐
│ Node.js │  │  Skynet  │
│         │  │  + TSTL  │
└────┬────┘  └────┬─────┘
     │            │
     ▼            ▼
 Promise      Coroutine
```

#### 实现细节

**Step 1: TSTL 配置**

`tsconfig.tstl.json`:
```json
{
  "tstl": {
    "luaTarget": "5.4",
    "luaLibImport": "require",
    "sourceMapTraceback": true
  }
}
```

**Step 2: TypeScript 代码**

```typescript
// 业务代码
export class LoginService {
  async handleLogin(request: LoginRequest): Promise<LoginResponse> {
    // 异步调用
    await runtime.timer.sleep(100);
    
    const result = await runtime.network.call(
      'database',
      'lua',
      'query',
      'SELECT * FROM users WHERE username = ?',
      request.username
    );
    
    return { success: true, user: result };
  }
}
```

**Step 3: TSTL 转换过程**

TSTL 会将上述代码转换为类似这样的 Lua:

```lua
-- 简化的转换结果
function LoginService:handleLogin(request)
  -- TSTL 生成的协程包装
  return __TS__AsyncAwaiter(function()
    -- await runtime.timer.sleep(100)
    -- 实际调用 skynet.sleep(10)，内部会 coroutine.yield
    coroutine.yield(skynet.sleep(10))
    
    -- await runtime.network.call(...)
    -- 实际调用 skynet.call，内部也会 coroutine.yield
    local result = skynet.call('database', 'lua', 'query', 
                               'SELECT * FROM users WHERE username = ?', 
                               request.username)
    
    return {success = true, user = result}
  end)
end
```

**Step 4: Skynet 适配器实现**

参见 [`src/runtime/skynet-adapter.ts`](../src/runtime/skynet-adapter.ts)

```typescript
export class SkynetTimer implements ITimer {
  async sleep(ms: number): Promise<void> {
    const centiseconds = Math.floor(ms / 10);  // Skynet 用厘秒
    skynet.sleep(centiseconds);  // 这会被编译为直接调用，内部 yield
  }
}

export class SkynetNetwork implements INetwork {
  async call(address: string, messageType: string, ...args: any[]): Promise<any> {
    // skynet.call 内部会 yield 当前协程，等待响应后 resume
    const result = skynet.call(address, messageType, ...args);
    return result;
  }
}
```

### 3.3 关键技术点

#### ✅ 为什么能工作?

1. **TSTL 的魔法**:
   - TSTL 理解 `async/await` 语法
   - 自动生成协程包装代码
   - 将 `await` 转换为适当的 `coroutine.yield` 调用

2. **Skynet 的协程机制**:
   - `skynet.call()` 内部实现:
     ```lua
     function skynet.call(addr, typename, ...)
       local session = c.send(addr, typename, ...)
       -- 关键: yield 当前协程，等待响应
       return coroutine.yield("SUSPEND")
     end
     ```
   - 当响应到达时，Skynet 框架会自动 `coroutine.resume` 对应的协程

3. **无缝桥接**:
   ```
   TS async/await → TSTL 编译 → Lua coroutine → Skynet yield/resume
   ```

#### 📊 执行流程图

```
┌─────────────────────────────────────────────────────────┐
│ 业务代码: await network.call('db', 'lua', 'query', id) │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
            ┌────────────────────────┐
            │ TSTL 编译为 Lua        │
            └────────────┬───────────┘
                         │
                         ▼
            ┌────────────────────────┐
            │ skynet.call(...)       │
            │ 内部 yield 协程         │
            └────────────┬───────────┘
                         │
                         ▼
            ┌────────────────────────┐
            │ Skynet 发送请求        │
            │ 协程挂起，释放 CPU     │
            └────────────┬───────────┘
                         │
                  (等待响应...)
                         │
                         ▼
            ┌────────────────────────┐
            │ 响应到达               │
            │ Skynet resume 协程     │
            └────────────┬───────────┘
                         │
                         ▼
            ┌────────────────────────┐
            │ 协程继续执行           │
            │ 业务代码获得返回值      │
            └────────────────────────┘
```

## 4. 业务代码示例

### 4.1 登录服务完整示例

参见 [`src/examples/login-service.ts`](../src/examples/login-service.ts)

#### TypeScript 代码

```typescript
import { runtime } from '../core/interfaces';

export interface LoginRequest {
  username: string;
  password: string;
}

export interface LoginResponse {
  success: boolean;
  user?: User;
  error?: string;
}

export class LoginService {
  private users = new Map<number, User>();
  
  /**
   * 初始化服务
   */
  async init(): Promise<void> {
    runtime.logger.info('LoginService initializing...');
    
    // 注册消息处理器
    runtime.network.dispatch('lua', async (session, source, cmd, ...args) => {
      if (cmd === 'login') {
        const [username, password] = args;
        const response = await this.handleLogin({ username, password });
        runtime.network.ret(response.success, response.user, response.error);
      } else if (cmd === 'logout') {
        const [userId] = args;
        const success = await this.handleLogout(userId);
        runtime.network.ret(success);
      }
    });
    
    runtime.logger.info('LoginService initialized');
  }
  
  /**
   * 处理登录 - 演示异步操作
   */
  async handleLogin(request: LoginRequest): Promise<LoginResponse> {
    runtime.logger.info(`Login attempt: ${request.username}`);
    
    // 模拟异步数据库查询
    await runtime.timer.sleep(100);
    
    // 验证逻辑
    if (!request.username || !request.password) {
      return { success: false, error: 'Invalid credentials' };
    }
    
    // 可以调用其他服务
    // const dbService = await runtime.service.newService('database');
    // const userData = await runtime.network.call(dbService, 'lua', 'getUserByName', request.username);
    
    const user: User = {
      userId: this.nextUserId++,
      username: request.username,
      token: this.generateToken(request.username),
      loginTime: runtime.timer.now(),
    };
    
    this.users.set(user.userId, user);
    
    runtime.logger.info(`User logged in: ${user.username}`);
    
    return { success: true, user };
  }
  
  async handleLogout(userId: number): Promise<boolean> {
    const user = this.users.get(userId);
    if (!user) return false;
    
    this.users.delete(userId);
    runtime.logger.info(`User logged out: ${user.username}`);
    return true;
  }
  
  private generateToken(username: string): string {
    return `${username}_${runtime.timer.now()}_${Math.random()}`;
  }
}

/**
 * 服务启动入口
 */
export async function main(): Promise<void> {
  const service = new LoginService();
  
  runtime.service.start(async () => {
    runtime.logger.info('=== Login Service Starting ===');
    await service.init();
    runtime.logger.info('=== Login Service Ready ===');
  });
}
```

### 4.2 编译后的 Lua 代码（简化）

```lua
-- dist/lua/examples/login-service.lua (TSTL 生成，简化版)

local LoginService = {}
LoginService.__index = LoginService

function LoginService.new()
  local self = setmetatable({}, LoginService)
  self.users = {}
  self.nextUserId = 1
  return self
end

function LoginService:init()
  -- 这里会被 TSTL 包装为协程
  local runtime = require("core.interfaces").runtime
  
  runtime.logger:info("LoginService initializing...")
  
  runtime.network:dispatch("lua", function(session, source, cmd, ...)
    local args = {...}
    if cmd == "login" then
      local username, password = args[1], args[2]
      -- 调用异步方法，这里会 yield
      local response = self:handleLogin({username = username, password = password})
      runtime.network:ret(response.success, response.user, response.error)
    end
  end)
  
  runtime.logger:info("LoginService initialized")
end

function LoginService:handleLogin(request)
  local runtime = require("core.interfaces").runtime
  
  runtime.logger:info("Login attempt: " .. request.username)
  
  -- await sleep(100) 被转换为:
  skynet.sleep(10)  -- 100ms = 10 centiseconds
  
  if not request.username or not request.password then
    return {success = false, error = "Invalid credentials"}
  end
  
  local user = {
    userId = self.nextUserId,
    username = request.username,
    token = self:generateToken(request.username),
    loginTime = skynet.time(),
  }
  
  self.nextUserId = self.nextUserId + 1
  self.users[user.userId] = user
  
  runtime.logger:info("User logged in: " .. user.username)
  
  return {success = true, user = user}
end

return {LoginService = LoginService, main = main}
```

2. **异步模型桥接**
   - TS async/await → Lua coroutine
   - Promise → yield/resume
   - 无缝的异步调用体验

3. **运行时抽象**
   - 接口层完全解耦业务和平台
   - 一套代码双环境运行
   - 易于扩展新的运行时

### 🎯 实际应用价值

- **开发效率提升**: TypeScript 类型安全 + IDE 智能提示
- **测试便利性**: Node.js 快速验证，无需启动 Skynet
- **生产性能保证**: Skynet Actor 模型 + 高性能 C 核心
- **团队协作**: 统一的代码规范和接口约定
