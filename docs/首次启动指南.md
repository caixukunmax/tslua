# TS-Skynet 首次启动指南

## 🚀 快速开始（3 步）

### 第 1 步：运行启动脚本
```bash
./start.sh
```
这将打开交互式菜单界面。

### 第 2 步：选择「1. 一键启动」
脚本会自动：
- ✅ 检查 Skynet 引擎是否已编译
- ✅ 编译 TypeScript → Lua
- ✅ 启动 Skynet 服务

### 第 3 步：完成！
服务启动后，你会看到类似输出：
```
========================================
  Game Server Starting...
========================================
```

---

## 📋 常用操作

### 查看服务状态
```bash
./start.sh status
```

### 停止服务
```bash
./start.sh stop
```

### 热更新代码
```bash
./start.sh hotfix login    # 热更新登录服务
./start.sh hotfix all      # 热更新所有服务
```

### 查看日志
```bash
./start.sh logs
```

---

## 🔧 首次运行可能需要

### 如果 Skynet 引擎未编译
```bash
./scripts/build.sh engine
```
这可能需要几分钟时间。

### 如果 npm 依赖未安装
```bash
npm install
```

## 📁 项目结构

```
tslua/
├── src/              # TypeScript 源码
├── dist/lua/         # 编译后的 Lua 代码 ✅
├── config/           # 配置文件
├── skynet/           # Skynet 框架
│   ├── skynet        # Skynet 引擎 ✅
│   └── config.tslua  # 配置文件（指向 dist/lua/）
└── logs/             # 运行日志
```

---

## ⚙️ 工作原理

1. **编译**：TypeScript 源码 → `dist/lua/`
2. **配置**：`skynet/config.tslua` 指向 `../dist/lua/`
3. **启动**：Skynet 读取配置，加载 Lua 代码

**无需手动复制文件！** 配置文件已经设置好了路径。

## 🆘 遇到问题？

### 问题：找不到 `./start.sh` 命令
**解决**：确保有执行权限
```bash
chmod +x ./start.sh
```

### 问题：Skynet 引擎未编译
**解决**：
```bash
./scripts/build.sh engine
```

### 问题：端口已被占用
**解决**：停止旧进程
```bash
./start.sh stop
./start.sh start
```

### 问题：热更新不生效
**解决**：确保服务正在运行
```bash
./start.sh status   # 检查
./start.sh hotfix all  # 热更新
```
