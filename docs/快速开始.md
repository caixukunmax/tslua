# 快速开始指南

> 10 分钟搭建你的第一个 TS-Skynet 混合服务

---

## 前置要求

- Node.js 20+
- Linux/macOS (Skynet 需要)
- 基础的 TypeScript 知识

---

## Step 1: 安装依赖

```bash
cd /mnt/d/tslua
npm install
```

预期输出:
```
added 234 packages in 15s
```

---

## Step 2: 构建项目

```bash
chmod +x build.sh
./build.sh
```


## Step 4: 编译 Skynet (首次需要)

```bash
cd skynet
make linux
```

等待几分钟编译完成。

---

## Step 5: 配置 Skynet 服务

创建配置文件 `skynet/examples/config.tskynet`:

```lua
-- Skynet 配置
thread = 8
harbor = 0
start = "main"                    -- 启动脚本
bootstrap = "snlua bootstrap"

-- 服务路径配置
luaservice = "./service/?.lua;./service-ts/?.lua"
lua_path = "./lualib/?.lua;./lualib/?/init.lua;./service-ts/?.lua"
lua_cpath = "./luaclib/?.so"

-- 日志配置
logger = "logger"
logservice = "snlua"
logpath = "."
```

---

## Step 6: 创建 Skynet 启动脚本

创建 `skynet/service-ts/main.lua`:

```lua
-- 加载编译后的 TS 代码
local skynet = require "skynet"

skynet.start(function()
    skynet.error("=== TS-Skynet Service Starting ===")
    
    -- 这里加载你编译后的 TypeScript 服务
    -- 由于 TSTL 的模块系统,需要手动适配
    skynet.error("Service ready")
    
    skynet.exit()
end)
```

---

## Step 7: 运行 Skynet

```bash
cd skynet
./skynet examples/config.tskynet
```

---

## 常见问题

### Q1: 编译失败 "Cannot find module 'typescript'"

**解决**:
```bash
npm install
```

### Q2: Skynet 编译错误

**解决**:
```bash
cd skynet
make clean
make linux
```

### Q3: Node.js 运行报错

**解决**: 检查 Node.js 版本
```bash
node --version  # 应该 >= 20.0.0
```
